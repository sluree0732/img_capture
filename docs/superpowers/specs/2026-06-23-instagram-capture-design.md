# 인스타그램 게시물 캡처 도구 — 설계 문서

## 배경 및 목적

대성테크윈의 인스타그램 계정에 올린 카드뉴스 게시물을 과거 업로드 순서대로 화면 캡처하여
`{접두어}_인스타_카드뉴스{NN}.png` 형식의 순차 파일로 저장하는 자사 콘텐츠 백업 도구.
최종적으로 GUI를 가진 단일 exe로 패키징하여, 비개발자도 입력값만 넣고 실행할 수 있게 한다.

## 범위 (v1)

**포함:**
- 인스타그램만 지원 (페이스북은 v2 이후)
- 이미지 1장으로 캡처가 끝나는 게시물만 지원 (본문이 길어 스크롤이 필요한 경우는 v1 범위 제외)
- 매 실행마다 ID/PW로 직접 로그인 (세션/쿠키 재사용 없음)
- 매 실행마다 전체 게시물을 처음부터 다시 캡처 (증분 캡처 없음)

**v1 범위 제외 (향후 확장):**
- 페이스북 지원
- 본문이 길어 스크롤이 필요한 게시물의 다중 캡처 (`_1`, `_2`... 분할 저장)
- 로그인 세션(쿠키) 재사용
- 이전 실행 결과를 기억해서 새 게시물만 캡처하는 증분 모드

## 기술 스택

| 항목 | 선택 | 이유 |
|---|---|---|
| 브라우저 자동화 | Selenium + webdriver-manager | 검증된 사례가 가장 많고, 막히는 부분이 생겼을 때 참고할 자료가 풍부함 |
| GUI | Tkinter | Python 기본 내장, exe 패키징 시 의존성/용량 부담이 적음 |
| 패키징 | PyInstaller (`--onefile --noconsole`) | 단일 exe 파일 생성, 콘솔 창 미표시 |
| 이미지 캡처 | Selenium 4 `WebElement.screenshot()` | 특정 요소 영역만 잘라서 캡처 가능 |

대상 PC에 Chrome이 설치되어 있어야 한다 (Selenium/webdriver-manager 공통 전제).

## 파일 구조

```
img_capture/
├── main.py        # Tkinter GUI, 입력값 수집, 덮어쓰기 확인 팝업, 전체 실행 오케스트레이션
├── auth.py        # 로그인 자동 입력 + 2FA/캡차 대기 처리
├── collector.py   # 게시물 그리드 스크롤하며 링크 수집 → 과거순 정렬
├── capture.py     # 게시물 진입 → 영역 스크린샷 → 파일 저장
└── config.py      # 셀렉터, 경로, 타임아웃 등 상수
```

## 상세 흐름

### 1. GUI 입력 및 실행 트리거 (`main.py`)
입력 필드: 아이디 / 비밀번호 / 게시물(프로필) URL / 저장 경로(폴더 선택) / 파일명 접두어.
[실행하기] 클릭 시:
1. 입력값 유효성 확인 (빈 값 체크)
2. 저장 경로에 `{접두어}_인스타_카드뉴스*.png` 패턴과 일치하는 파일이 이미 존재하는지 확인
   - 존재하면 팝업: "이미 캡처된 파일이 있습니다. 계속하면 덮어씁니다. 진행하시겠습니까?" (예/아니오)
   - 아니오 선택 시 → 브라우저를 띄우지 않고 작업 중단
3. 진행 → 아래 2~5단계 순차 실행, 진행 상황("N/M 캡처 중")과 로그를 GUI에 표시

### 2. 로그인 (`auth.py`)
1. Chrome 실행, 인스타그램 로그인 페이지 이동
2. GUI에서 받은 ID/PW를 입력 필드에 자동 입력 후 로그인 버튼 클릭
3. 로그인 성공 여부를 주기적으로 확인 (URL 변화 등으로 판별, 타임아웃 설정)
4. 2FA 인증 화면 또는 캡차로 판단되는 화면이 감지되면:
   - 자동화를 일시정지하고 GUI에 "사용자 확인이 필요합니다. 브라우저에서 직접 처리해주세요" 안내 표시
   - 사용자가 브라우저에서 해당 단계를 직접 완료하면, 프로그램이 로그인 성공을 감지하여 자동으로 다음 단계 진행
   - 일정 시간 내 로그인 성공이 감지되지 않으면 타임아웃 처리 (작업 중단 + 로그 기록)

### 3. 게시물 수집 (`collector.py`)
1. 입력받은 프로필/게시물 목록 URL로 이동
2. 게시물 그리드를 아래로 스크롤하며 게시물 링크를 누적 수집
   - 인스타그램은 최신 → 과거 순으로 로드되므로, 스크롤해도 더 이상 새 항목이 로드되지 않을 때까지 반복
3. 수집된 링크 리스트를 뒤집어 과거 → 최신 순으로 정렬하여 반환

### 4. 캡처 및 저장 (`capture.py`)
정렬된 순서대로 각 게시물에 대해:
1. 게시물 링크로 이동, 상세 패널 로딩 대기
2. 게시물 영역(이미지+본문+해시태그가 포함된 패널) 요소만 스크린샷
3. `{접두어}_인스타_카드뉴스{NN}.png` (NN = 01부터 2자리 0패딩, 순서대로 증가)로 저장
4. 해당 게시물 처리 중 오류(로딩 실패 등) 발생 시 → 건너뛰고 로그에 기록, 다음 게시물 계속 진행

### 5. 완료
모든 게시물 처리 후 GUI에 "완료, N개 캡처됨" 표시.

## 예외 처리 정리

| 상황 | 처리 |
|---|---|
| 네트워크/페이지 로딩 실패 (개별 게시물) | 해당 게시물 건너뛰고 로그 기록, 전체 작업은 계속 |
| 2FA/캡차 대기 타임아웃 | 알림 후 작업 중단, 로그 기록 |
| 저장 경로에 동일 파일명 존재 | 실행 전 팝업으로 계속/취소 확인 |
| 입력값 누락 | 실행 버튼 클릭 시 즉시 안내 메시지, 진행 안 함 |

## 리스크 및 완화

- **로그인 자동화에 대한 봇 탐지/캡차**: 완전 무인 자동화는 보장할 수 없음. 캡차/2FA 발생 시 사용자가 직접 개입하는 반자동 구조로 설계하여 대응.
- **계정 보안 정책 위반 가능성**: 자사 소유 계정의 콘텐츠 백업 목적이며, 캡차/탐지 우회 기법(지문 위장, 자동 풀이 서비스 등)은 적용하지 않음. 자주 실행할 경우 보안 경고 빈도가 늘어날 수 있으며, 필요 시 추후 세션 재사용 기능을 v2에서 추가 검토.
- **Chrome 버전 변경**: webdriver-manager가 실행 시점에 맞는 chromedriver를 자동으로 받아오므로 일반적인 Chrome 업데이트에는 대응되나, 사내 네트워크에서 외부 다운로드가 차단된 환경이면 실행이 실패할 수 있음.
- **인스타그램 DOM 구조 변경**: 셀렉터가 깨지면 유지보수 필요. `config.py`에 셀렉터를 모아두어 변경 시 한 곳만 수정하면 되도록 설계.
